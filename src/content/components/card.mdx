# Card

## Base Component

**Default Behavior**

A card is an enclosed container that has special rules for padding that solve optical asymmetries in certain scenarios. It supports an optional state layer and background scrim. The scrim is absolutely positioned to take up the full height and width, acting as a background controller that can layer solid fills, opacity effects, and backdrop filters. The background color of the card itself is never directly modifiedâ€”only the border is. Overflow is hidden by default.

### HTML

```html
<div lk-component="card" lk-card-scale-factor="body" lk-card-variant="fill" lk-card-material="flat">
  <div lk-component="slot" lk-slot="children"></div>
</div>
```

### TypeScript

```tsx
import { useMemo } from "react";
import { propsToDataAttrs } from "../utilities";

interface LkCardProps extends React.HTMLAttributes<HTMLDivElement> {
  scaleFactor?: LkFontClass;
  variant?: "fill" | "outline" | "transparent";
  material?: "flat" | "glass" | "rubber";
  opticalCorrection?: "top" | "left" | "right" | "bottom" | "x" | "y" | "all" | "none";
  isClickable?: boolean;
  children?: React.ReactNode;
}

export default function Card(props: LkCardProps) {
  const { children, ...restProps } = props;

  const lkCardAttrs = useMemo(
    () => propsToDataAttrs(restProps, "lk-card"),
    [restProps.scaleFactor, restProps.variant, restProps.material, restProps.opticalCorrection, restProps.isClickable]
  );

  return (
    <div {...lkCardAttrs}>
      <div lk-component="slot" lk-slot="children">
        {children}
      </div>
    </div>
  );
}
```

## Combine with Native HTML Attributes

You can set any other _native_ HTML attribute (className, style, id, etc.) on the component by passing it as a prop.

## Props

### **scaleFactor**

**Description**

Controls padding and border radius using a complex set of optical correction rules. This prop should match the largest font size used within the card to ensure that spacing and visual rhythm remain balanced.

| **Expected Type** | **Default Value** | **Acceptable Values** |
| ----------------- | ----------------- | --------------------- |
| `LkFontClass`     | `body`            | Any `LkFontClass`     |

```tsx
<Card scaleFactor="display1" />
```

### **variant**

**Description**

Controls the background behavior of the card using a separate scrim layer.

- `fill`: Applies a solid background fill, typically `lk-surfacecontainerlowest`.
- `outline`: Makes the background transparent and applies a subtle border.
- `transparent`: Removes both background fill and border entirely.

| **Expected Type** | **Default Value** | **Acceptable Values**            |
| ----------------- | ----------------- | -------------------------------- |
| `string`          | `fill`            | `fill`, `outline`, `transparent` |

```tsx
<Card variant="outline" />
```

### **material**

**Description**

Controls the card's backdrop-filter behavior:

- `flat`: No backdrop filters are applied.
- `glass`: Adds a blur effect equivalent to the defined `lk-material-prop-thickness`.
- `rubber`: Reserved for neumorphic styles (empty ruleset for now).

| **Expected Type** | **Default Value** | **Acceptable Values**     |
| ----------------- | ----------------- | ------------------------- |
| `string`          | `flat`            | `flat`, `glass`, `rubber` |

```tsx
<Card material="glass" />
```

### **opticalCorrection**

**Description**

Fine-tunes the internal offset adjustments used for optical corrections. Useful when aligning elements more precisely along specific axes or directions.

| **Expected Type** | **Default Value** | **Acceptable Values**                                     |
| ----------------- | ----------------- | --------------------------------------------------------- |
| `string`          | `top`             | `top`, `left`, `right`, `bottom`, `x`, `y`, `all`, `none` |

```tsx
<Card opticalCorrection="y" />
```

### **isClickable**

**Description**

Indicates whether the card should respond to hover/focus interactions. When `true`, a state layer is shown, making it behave more like a button. But you will need to add the click handler yourself.

| **Expected Type** | **Default Value** | **Acceptable Values** |
| ----------------- | ----------------- | --------------------- |
| `boolean`         | `false`           | `true`, `false`       |

```tsx
<Card isClickable={true} />
```
