## Base Component

**Default Behavior**

A dropdown is a relatively positioned container with two children: a `DropdownChip` and a `DropdownList`. The parent container provides positioning context for the list, which can be absolutely positioned relative to the chip. The dropdown itself has no other defining styles or interactivityâ€”it is purely a structural wrapper.

### HTML

```html
<div lk-component="dropdown">
  <div lk-component="dropdown-chip">Select an option</div>
  <div lk-component="dropdown-list">
    <div class="dropdown-item">Option 1</div>
    <div class="dropdown-item">Option 2</div>
    <div class="dropdown-item">Option 3</div>
  </div>
</div>
```

### TypeScript

```tsx
import { useMemo } from "react";
import { propsToDataAttrs } from "../utilities";

interface LkDropdownProps extends React.HTMLAttributes<HTMLDivElement> {
  children?: React.ReactNode;
}

export default function Dropdown(props: LkDropdownProps) {
  const { children, ...restProps } = props;

  const dropdownAttrs = useMemo(() => propsToDataAttrs(restProps, "lk-dropdown"), []);

  return <div {...dropdownAttrs}>{children}</div>;
}
```

### CSS (Reference)

```css
[lk-component="dropdown"] {
  position: relative;
}
```

## Combine with Native HTML Attributes

You can set any other _native_ HTML attribute (className, style, id, etc.) on the component by passing it as a prop.

## Props

This component does not have any custom LiftKit props. All logic and UI is delegated to its child components: `DropdownChip` and `DropdownList`.
